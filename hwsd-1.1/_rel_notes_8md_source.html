6 # Introduction {#Introduction}
7 
8 Welcome to HWSD, a daemon and library for the discovery and announcement
9 of hardware resources using ZeroConf. HWSD enables auto-configuration of
10 ad-hoc GPU clusters and multi-GPU machines.
11 
12 HWSD 1.0 is a feature release based on GPU-SD 1.4. It can be retrieved by
13 downloading the
14 [source code](http://www.equalizergraphics.com/gpu-sd/downloads/hwsd-1.0.0.tar.gz)
15 or one of the
16 [binary distributions](http://www.equalizergraphics.com/gpu-sd/downloads.html).
17 
18 ## Features {#Features}
19 
20 HWSD provides the following features to enable GPU and network device discovery:
21 
22 * Local Discovery: Automatic network device and GPU discovery for Linux
23  (X11/GLX), Mac OS X (CGL, GLX) and Windows (WGL, WGL_NV_gpu_affinity,
24  WGL_AMD_gpu_association).
25 * Network announcement: A daemon for network announcement of available
26  devices and their configuration using DNS service discovery and ZeroConf
27  networking.
28 * Remote Discovery: Remote GPU discovery of resources announced using
29  the HWSD daemon.

78 # About {#About}
79 
80 HWSD is a cross-platform library, designed to run on any modern
81 operating system, including all Unix variants and the Windows operating
82 system. Zeroconf support in Lunchbox is required for the DNS_SD module
83 and applications.
84 
85 HWSD uses CMake and [Buildyard](https://github.com/Eyescale/Buildyard)
86 to create a platform-specific build environment. The following platforms
87 and build environments are tested:
88 
89 * Linux: Ubuntu 12.04, 12.10, 13.04, RHEL 6.3 (Makefile, i386, x64)
90 * Windows: 7 (Visual Studio 2008, i386, x64)
91 * Mac OS X: 10.8 (Makefile, i386, x64)
