1 # CMake Modules
2 
3 This repository contains common CMake modules. To use it, create a
4 .gitexternals in your project:
5 
6  # -*- mode: cmake -*-
7  # CMake/common https://github.com/Eyescale/CMake.git master
8 
9 Copy [GitExternal](GitExternal.cmake) from this repository to CMake/,
10 and use it in your top-level CMakeLists.txt:
11 
12  list(APPEND CMAKE_MODULE_PATH ${PROJECT_SOURCE_DIR}/CMake
13  ${PROJECT_SOURCE_DIR}/CMake/common)
14  include(GitExternal)
15  include(Common)
16 
17 ## Documentation
18 
19 * [Common](Common.cmake) does a common CMake setup, including:
20  * [CommonLibrary](CommonLibrary.cmake) *common_library* function to
21  build a shared library using a standard recipe.
22  * [CommonApplication](CommonApplication.cmake) *common_application*
23  function to build an application using a standard recipe.
24  * [CommonCode](CommonCode.cmake) generates standard header files for
25  the project (api.h, defines.h, version.h).
26  * **GNUModules**: *module* target to create a
27  [GNUModules](GNUModules.cmake) *module* and *snapshot* targets to
28  create [GNU modules](http://modules.sourceforge.net/).
29  * [GitTargets](GitTargets.cmake) *branch*, *cut*, *tag*, *erase*,
30  *retag*, *tarball* targets.
31  * [Compiler](Compiler.cmake): Default compiler flags, useful default
32  warnings and 'safe' C++11 features.
33  * [GitInfo](GitInfo.cmake) sets variables with information about the
34  git source tree.
35 * [CommonCTest](CommonCTest.cmake) does a common CTest setup, including
36  * Automatically adding all .cpp files as tests
37  * [Coverage](Coverage.cmake) Create code coverage report as html, if
38  ENABLE_COVERAGE is set. Buildyard has 'make Coverage' target to
39  enable this in a separate build, since coverage flags may break
40  downstream projects.
41  * [CppcheckTargets](CppcheckTargets.cmake): *cppcheck* target for
42  static code analysis. Also adds all cppcheck targets to tests.
43  * [clangcheckTargets](clangcheckTargets.cmake): *clangcheck* target for
44  clang-check code analysis. Adds all cppcheck targets to tests if
45  ENABLE_CLANGCHECK_TESTS is set.
46 * [DoxygenRule](DoxygenRule.cmake): *doxygen* target to build
47  documentation into PROJECT_BINARY_DIR/doc. Optional *doxygit* target to
48  copy result to ../GITHUB_ORGANIZATION/Project-M.m/. Must be included
49  after all targets.
50 * [SubProject](SubProject.cmake): Using a top level simple CMake
51  superproject, several CMake subprojects (which may depend on each other)
52  can be build. Each subproject can be added to the superproject
53  with an add_subdirectory( *subproject* ) directive. To use the SubProject
54  feature, the sub projects should modify their CMake scripts.
55  In the scripts CMAKE_BINARY_DIR should be changed to PROJECT_BINARY_DIR
56  and CMAKE_SOURCE_DIR should be changed to PROJECT_SOURCE_DIR. A
57  sample project can be found at https://github.com/bilgili/SubProjects.git
58 
59 * Find scripts for non-CMake projects.
